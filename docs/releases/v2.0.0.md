# Harmony v2.0.0 Release Notes

**Release date:** 2026-07-24  
**Upgrade path:** v1.5.0 → v2.0.0

Harmony v2.0.0 turns the Library foundation introduced in v1.5.0 into a
complete music-operations platform. It adds review-first metadata repair,
durable download operations, direct Navidrome playlist synchronization,
playlist file management, automatic playlists and source schedules, and a
mobile-first interface polish pass.

Harmony v1.6.0 was never published. Earlier documentation that described these
changes as v1.6.0 was premature; v2.0.0 is the next release after v1.5.0 and
contains that work plus all subsequent improvements.

## Highlights

### Metadata Intelligence

- Analyze indexed Songs with durable, provider-neutral metadata-health rules.
- Discover MusicBrainz candidates with bounded searches, explainable scoring,
  confidence levels, and explicit ambiguity confirmation.
- Review and accept individual field suggestions before changing canonical
  Library metadata.
- Preview, apply, audit, and roll back supported metadata changes through
  durable background jobs.
- Explicitly write canonical metadata and cached artwork to audio-file tags;
  discovery and canonical application never modify files silently.
- Optionally enrich genres from Spotify artist metadata without making Spotify
  failures block downloads.

### Downloads Operations Center

- Inspect active work, the deterministic queue, persisted history, telemetry,
  outcomes, retry counts, and safe job details.
- Retry failed or cancelled jobs, cancel active jobs, and clear selected or
  categorized history without deleting music or Library records.
- Download from Spotify and, when explicitly enabled, public YouTube Music or
  YouTube URLs through yt-dlp.
- Recover more safely from cancellation, deleted-file redownloads, provider
  failures, and legacy database states.

### Navidrome and Playlist Operations

- Check Navidrome connectivity and scanner state and request incremental or
  full rescans from Harmony.
- Synchronize playlist song IDs and ordering directly through Navidrome's
  Subsonic API.
- Fall back safely to targeted M3U export/import when a direct update is
  ambiguous, read-only, or unsuccessful.
- Inspect playlist availability, filter playlists, download M3U files, manage
  playlist Library files, and delete saved playlist definitions without
  deleting downloaded Songs.
- Preserve source ordering and export only files that currently exist.

### Automation

- Generate 50-song Recently Added and Recently Downloaded auto-playlists, with
  a configurable 1–500-song limit.
- Enable per-source auto-sync schedules of hourly, every 6 hours, every 12
  hours, daily, or weekly.
- Keep auto-playlists and affected exports current after relevant downloads and
  Library changes.

### Mobile and Accessibility

- Replace the fixed mobile sidebar with compact navigation suited to small
  screens.
- Redesign Settings section discovery and polish Dashboard, Downloads,
  Sources, Library tabs, overlays, dialogs, pagination, and dense action rows.
- Improve labels, focus behavior, tap targets, scrolling, overflow handling,
  artwork presentation, and reduced-motion support.

## Upgrade

1. Back up Harmony's persistent SQLite database, music directory, playlist
   directory, artwork cache, and environment file.
2. Stop the v1.5.0 deployment.
3. Deploy the v2.0.0 image or rebuild from the v2.0.0 source.
4. Review `.env.example`, especially Navidrome, MusicBrainz, Spotify genre, and
   YouTube Music settings.
5. Start Harmony normally. Alembic applies database revisions
   `20260721_0009` through `20260724_0025` automatically.
6. Allow the initial Library reconciliation to finish before starting large
   metadata, playlist, or bulk operations.

Do not attempt to install an intermediate v1.6.0 release. There is no published
v1.6.0 upgrade step.

## Validate the Deployment

Check service health:

```bash
curl http://localhost:8000/health
```

The response should report `"status":"ok"` and `"version":"2.0.0"`.

Then verify:

1. Dashboard and Library counts load without an error.
2. Library Health can run a metadata analysis.
3. Downloads shows queue/history snapshots and opens job details.
4. Navidrome status is correct for the configured server.
5. A Source can save an auto-sync interval and retain it after refresh.
6. Auto-playlist generation produces the configured bounded result.
7. A playlist M3U download contains only available files in source order.

## Compatibility and Known Limits

- Python 3.12 remains the supported runtime.
- SQLite remains the only supported database.
- Metadata discovery and application are public for Songs; Album and Artist
  matching remain internal foundations.
- YouTube Music support is opt-in and depends on public yt-dlp extraction. It
  does not bypass regional, authentication, age, removal, or rate limits.
- Direct playlist sync targets Navidrome. Other media servers continue to use
  M3U exports.
- Duplicate grouping, favorites/ratings, user-defined smart rules, API
  authentication, and multi-user support are not part of v2.0.0.
