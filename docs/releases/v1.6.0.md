# Harmony v1.6.0 Release Notes

**Release date:** 2026-07-22

Harmony v1.6.0 delivers **Metadata Intelligence**, a review-first workflow for
improving the Library Index without silently changing your music metadata. It
adds provider-neutral metadata health analysis, MusicBrainz-backed discovery,
explainable candidate matching, reviewed suggestions, durable background
application, audit history, and reversible rollback.

## Highlights

- **Metadata Health:** The Library Health experience records durable issues for
  missing, inconsistent, and malformed metadata. Each issue includes severity,
  evidence, status, and repair guidance.
- **MusicBrainz Discovery:** Harmony searches MusicBrainz through a
  rate-limited, retry-aware, locally cached provider integration. Discovery
  retains a bounded set of candidates and their search provenance.
- **Explainable Matching:** Candidates are scored deterministically using
  title, artist, album, duration, identifiers, and version markers. Positive,
  conflicting, and unavailable evidence is retained for review. Ambiguous and
  low-confidence choices require explicit confirmation.
- **Review Before Apply:** Discovery produces per-field suggestions with
  confidence, provider provenance, evidence, and a canonical snapshot. You
  accept or reject suggestions before applying them.
- **Safe, Durable Application:** Preview accepted or selected changes, then
  run their application as a durable background batch. Per-Song reservations,
  stale-value checks, force-confirmation controls, progress, and structured
  outcomes protect concurrent and long-running operations.
- **Audit and Rollback:** Applied changes appear in metadata history with
  provider and job provenance. Reversible history entries and completed batches
  can be previewed and rolled back.

The initial public discovery workflow supports **Songs**. Album and Artist
matching is retained as an internal foundation for future releases.

## Upgrade

1. Back up your persistent Harmony SQLite database before deployment.
2. Deploy the v1.6.0 image or rebuild from this release.
3. Start Harmony normally. Alembic automatically applies revisions
   `20260721_0009` through `20260722_0015`, which add the metadata intelligence
   schema, provider cache, discovery and application records, and new indexed
   Song metadata fields.
4. Review `.env.example` if you need to tune MusicBrainz behavior. Defaults
   intentionally limit public-provider traffic to one request per second, with
   two concurrent requests and a 24-hour cache TTL.
5. Reload the Library and Library Health pages after the service is healthy.

For Docker Compose deployments:

```bash
docker compose up -d --build
docker compose logs -f harmony
```

## Validate the Deployment

Check the service version:

```bash
docker compose exec harmony curl -fsS http://127.0.0.1:8080/health
```

The response should report `"status":"ok"` and `"version":"1.6.0"`.

Confirm the metadata discovery boundary is available:

```bash
docker compose exec harmony curl -fsS \
  http://127.0.0.1:8080/api/metadata/discoveries/capabilities
```

The response should identify `musicbrainz`-compatible discovery capabilities
and list `song` as a public entity type. Provider diagnostics are also
available in the web UI and at `/api/providers/status`.

## Compatibility and Known Limits

- Harmony does not silently write provider values to audio files. A user must
  review and accept suggestions before they are applied.
- MusicBrainz is the initial discovery provider. Cover Art Archive ingestion,
  additional providers, and manual artwork replacement remain future work.
- Discovery and application are publicly available for Songs only in v1.6.0.
  Album and Artist projections support internal matching but are not public
  discovery targets yet.
- Existing M3U export remains Harmony's integration path for Navidrome,
  Jellyfin, Plex, and other compatible servers. Direct Navidrome
  synchronization is not part of this release.
