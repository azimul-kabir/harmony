# Harmony v1.5.0 Release Notes

**Release date:** 2026-07-21

Harmony v1.5.0 introduces the **Library Foundation**: a persistent, scalable
catalog for managed music and the user-facing tools built on top of it. Harmony
continues to acquire and organize music while Navidrome, Jellyfin, Plex, and
other compatible servers handle playback.

> Release history note: the next published release is
> [v2.0.0](v2.0.0.md). Harmony v1.6.0 was never published.

## Highlights

- **Persistent Library Index:** Stable Song IDs and indexed descriptive,
  technical, provenance, artwork, availability, and timestamp data.
- **Incremental Library Watcher:** Filesystem changes are reconciled without
  routine full-library scans, including additions, modifications, moves,
  renames, and missing files.
- **Fast Library Discovery:** SQLite FTS5 search, combined filters, stable
  sorting, responsive pagination, and Songs, Albums, Artists, and Collections
  views all read from the index.
- **Artwork Foundation:** Embedded and folder artwork are deduplicated in a
  content-addressed local cache and served through Library APIs.
- **Library Operations:** Analytics, health checks, maintenance actions, and
  task-backed bulk delete, move, rename, metadata refresh, artwork refresh,
  and ZIP export.

## Reliability Improvements

- The upgrade backfills legacy rows whose `songs.created_at` value is missing.
- Library Song API serialization now uses indexing, modification, or filesystem
  timestamps as safe fallbacks during an upgrade. This prevents old records
  from causing an invalid Library API response.

## Upgrade

1. Back up your persistent Harmony database before deployment.
2. Deploy the v1.5.0 image or rebuild from this release.
3. Start Harmony normally. Alembic applies the Library migrations and backfills
   missing creation timestamps automatically.
4. Reload the Library page after the service is healthy. A manual rescan is
   optional and is only needed when you want to refresh existing file metadata.

For Docker Compose deployments:

```bash
docker compose up -d --build
docker compose logs -f harmony
```

Verify the Library API once the container is running:

```bash
docker compose exec harmony curl -i \
  "http://127.0.0.1:8080/api/library/songs?limit=10"
```

The endpoint should return `HTTP/1.1 200 OK`.

## Compatibility and Known Limits

- Existing M3U export remains the integration path for Navidrome and other
  compatible servers. Direct Navidrome synchronization is not part of v1.5.0.
- Duplicate Detection is displayed as unavailable until a detection engine is
  implemented; it does not report a zero-duplicate result.
- Remote artwork providers, manual artwork replacement, metadata repair, and
  favorites remain planned work.
